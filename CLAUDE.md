## What This App Is

A single-page HTML dashboard (`index.html`) served via GitHub Pages.
No build step. All JS/CSS inline. Reads/writes to this repo via the GitHub Contents API.

**Live flow:** User logs in with a GitHub PAT → app reads `program.md` + `Tracker.md` → AI generates today's workout brief → user logs exercises set-by-set → confirm screen → commit appended entry to `Tracker.md`.

---

## Key Architecture Decisions

### Auth
- GitHub PAT stored in `localStorage`. Validated against `api.github.com/user` to get username + avatar (feels like OAuth without a server).
- No GitHub OAuth Device Flow — CORS limitations on the token endpoint make it impractical from the browser.

### AI Keys
- Never committed. Baked into `config.js` at deploy time by GitHub Actions using repo secrets (`GEMINI_API_KEY`, `ANTHROPIC_API_KEY`).
- `window.WORKOUT_CONFIG` set by `config.js`; app checks it before `localStorage` fallback.
- `getGeminiKey()` / `getClaudeKey()` / `getAIProvider()` helpers abstract the source.

### AI Brief as Source of Truth
- `loadAIBrief()` generates (or loads from `brief-cache.json`) today's workout brief.
- `renderBriefHTML()` sets the global `currentBrief = data`.
- `renderHome()` silently pre-loads `brief-cache.json` into `currentBrief` on startup.
- When the user opens the workout log screen, `renderWorkout()` checks `currentBrief?.exercises`.
  - If AI provided exercises → `wkExercises` is built from them (AI-sourced flag set).
  - If not → falls back to static `HYP_A` / `HYP_B` arrays.
- `buildStructuredMarkdown()` and `buildStructuredSummary()` both iterate `wkExercises` (not hardcoded arrays).
- The session title in the markdown header uses `currentBrief.sessionTitle` if available.

### Cross-Device Brief Cache
- `brief-cache.json` committed to the repo via GitHub API (not gitignored).
- Keyed by `{ date, dow }`. If today's cache exists, `renderBriefHTML` uses it.
- `saveBriefCache()` is fire-and-forget (doesn't block render).

### iOS Safari Compatibility
- All inputs use `font-size: 16px` minimum (prevents auto-zoom).
- `env(safe-area-inset-*)` for notch/home-bar padding.
- 44px minimum tap targets.
- `-webkit-backdrop-filter` for blur effects.
- `visualViewport` keyboard scroll handler.

## AI Brief JSON Schema

The AI prompt returns (and the app expects) this structure:

```json
{
  "sessionTitle": "string",
  "keyTargets": ["string"],
  "warnings": ["string"],
  "highlights": ["string"],
  "exercises": [
    { "name": "string", "sets": 3, "type": "strength|iso", "target": "string", "note": "string" }
  ],
  "cardio": {
    "prescription": "string",
    "paceTarget": "string",
    "hrTarget": "string",
    "notes": "string"
  }
}
```

`exercises[].sets` drives the number of set rows rendered in the workout form.
`exercises[].type === "iso"` switches the row to a time-based input instead of weight×reps.

## GitHub Actions Deploy

```yaml
- name: Generate config.js from secrets
  env:
    GEMINI_KEY: ${{ secrets.GEMINI_API_KEY }}
    CLAUDE_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
  run: |
    python3 << 'PYEOF'
    import os, json
    config = {
      "geminiKey": os.environ.get("GEMINI_KEY", ""),
      "claudeKey": os.environ.get("CLAUDE_KEY", ""),
    }
    with open("config.js", "w") as f:
      f.write("window.WORKOUT_CONFIG = " + json.dumps(config) + ";\n")
    PYEOF
```

The Pages source must be set to **GitHub Actions** (not "Deploy from branch") in repo Settings → Pages.

## Save Flow

1. User fills workout form OR types natural language description
2. "AI Parse & Preview" → `parseWithAI()` → AI formats markdown → shown in preview block
3. "Save Workout" → `confirmSave()` → shows confirm screen with what will be appended
4. If it's a `both` day (Tue/Sat) and user logged a lift → prompt to also log cardio
5. "Commit to Tracker" → `commitSave()` → GitHub Contents API PUT to `Tracker.md`

## Known Gotchas

- `brief-cache.json` may not exist on a fresh repo — `fetchGitHubFile` throws a 404, caught by `.catch(() => {})`.
- `btoa(unescape(encodeURIComponent(...)))` pattern required for UTF-8 → base64 for GitHub API.
- GitHub Pages must be configured to "GitHub Actions" source, not "Deploy from branch", for `config.js` to be included.
- The `wkData` keys must match the `id` field in `wkExercises`. AI-sourced exercises use `toSlug(name)` as the id.
